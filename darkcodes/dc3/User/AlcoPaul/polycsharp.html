High level language prepender basically does this


---------------------            -----------------------          ----------------------
      virus            infects            host            ----->          virus
---------------------            -----------------------          ----------------------
                                                                          host
                                                                  ----------------------

when the infected file is executed, the virus executes, does whatever it does and executes the host. simple.

let's make it not so simple.

towards polymorphism
--------------------

after reading "polymorphism in c#" by free0n which can be found in doomriderz #1, i learned of the coolness of 
.net's built-in class that allows us to programatically create an msil exe file from a source. we can put this into good use.

note: the checksum of each compilation of the same source is different. in other words, "polymorphic".

the approach towards polymorphism goes something like this

---------------------            -----------------------          ----------------------
      virus            infects            host            ----->         ViRuS(n)
---------------------            -----------------------          ----------------------
                                                                          host
                                                                  ----------------------
i reflected. and i somehow got an idea.



how can it be done?
-------------------

first, the virus reads itself, extracting its own source code. next, it modifies the prepared source code to add
poly routines (e.g. variable changing). then it calls the compiler classes to build an msil exe from the source code
, creating a different version of itself. finally, the virus copies the generated msil exe to the beginning part of the host.

remember that the generated msil exe is different (checksum and in its internal structure).
