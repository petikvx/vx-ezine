
Утилиты для "выдирания" бинарного кода из файла и дальнейшего его 
преобоазования.

ct.exe -
		Выдирает нужный код ,ограниченный сигнатурами "SBGN" и "SEND" -
		соответственно начало и конец.Код затем транслируется в инклуды.
		К инклудам добавляется заголовок из <out_name>.cgf (по 5 строк 
		для asm/c).Если конфига в текущей папке нет,то заголовок не 
		используется.

stospack.exe -
                Опять же данные из файла изымается ,но не  транслируется,а 
		создается asm файл ,содержащий исходник "распаковывающий" 
		данные по адресу,заданном параметром на стеке.Использован C 
		call.В некоторых случаях данные сжимаеются,но чаще - увеличивают
		свой объем.Помимо этого данные еще по ходу дела приобретают вид,
		который может быть и не таким как прежде ,т.е. если преобразовать
		данные утилитой 2 раза ,то не факт ,что "распаковывающий" код,
		для 1-го и 2-го случаев, будет идентичен для одних и тех же 
		данных.
